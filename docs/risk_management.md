# Risks & Contingencies

---

## 1. Scope Creep

### Risk
Too many agents, features, or systems introduced early.

### Mitigation
- Define strict MVP
- Limit initial agent set
- Add incrementally

---

## 2. Overengineering

### Risk
Applying complex architecture too early.

### Mitigation
- Start simple
- Avoid microservices
- Validate before scaling

---

## 3. Agent Overlap

### Risk
Agents duplicating responsibilities.

### Mitigation
- Enforce strict boundaries
- Define responsibilities clearly
- Review regularly

---

## 4. Lack of Clear Decisions

### Risk
Endless exploration without commitment.

### Mitigation
- Use decision logs
- Force explicit decisions
- Time-box exploration

---

## 5. Poor Output Quality

### Risk
Agents produce inconsistent or incorrect results.

### Mitigation
- Introduce validation layer early
- Use editor/validator agent
- Require human review initially

---

## 6. System Complexity Over Time

### Risk
System becomes difficult to maintain.

### Mitigation
- Maintain modular structure
- Avoid tight coupling
- Refactor regularly

---

## 7. Unclear Value

### Risk
System does not meaningfully improve workflow.

### Mitigation
- Define success criteria early
- Test against real use cases
- Iterate based on feedback

---

## 8. Time Constraints

### Risk
Project grows beyond available time.

### Mitigation
- Prioritise core functionality
- Treat as iterative project
- Cut non-essential features early
