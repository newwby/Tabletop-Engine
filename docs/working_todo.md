# Focus Areas (Current Exploration Phase)

The following areas define the **next stage of the project**. These are not yet fully defined, many representing decisions that need to be made, but are now the active focus for progression and working to-do list during the setup phase. Complexity should be avoided whilst core direction is still being defined (progress should be incremental and subject to review).

---

## 1. MVP / Core Direction

- [ ] Define MVP scope; minimum viable version of the system.
- [ ] Identify initial problems the system will solve.
- [ ] Establish system boundaries.

---

## 2. Architecture Decisions

- [ ] Identify key system architecture and design decisions from engineering principles.
- [ ] Group decisions into related logical issues.
- [ ] Evaluate trade-offs between approaches.
- [ ] Formalise decisions where confidence is sufficient.

---

## 3. Agents

- [ ] Define expectations for agent outputs.
- [ ] Establish agent definitions and initial agent layers.
- [ ] Establish agent constraints (e.g. over-generation, scope, continuity) and consistency.

---

## 4. Workflow Design

- [ ] Define how the system operates end-to-end:
  - [ ] How do agents interact at a high level?
  - [ ] How should data move between components and layers?
  - [ ] Where will validation and correction occur?
- [ ] Define staged development approach.
- [ ] Review README to ensure it reflects system truth.
- [ ] Review documentation directory.
