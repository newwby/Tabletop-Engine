## 1. Architecture Decision

Purpose:
- Explore and document key system decisions

Fields:
- Problem
- Options
- Trade-offs
- Constraints
- Open questions
- Decision (filled later)

---

## 2. Agent Definition

Purpose:
- Define a new agent clearly

Fields:
- Responsibility
- Inputs
- Outputs
- Boundaries (what it does NOT do)
- Dependencies

---

## 3. Agent Review

Purpose:
- Evaluate agent output quality

Fields:
- Expected behaviour
- Actual output
- Issues identified
- Suggested fixes
- Verdict

---

## 4. Task / Feature

Purpose:
- Track development work

Fields:
- Description
- Scope
- Acceptance criteria
- Dependencies

---

## 5. Bug

Purpose:
- Track issues in behaviour

Fields:
- Description
- Steps to reproduce
- Expected vs actual
- Severity

---

## 6. Research / Exploration

Purpose:
- Investigate ideas before committing

Fields:
- Topic
- Context
- Findings
- Relevance to project
- Next steps

---

## Notes

- Templates should minimise overhead
- Prefer fewer, high-quality issues over many low-value ones
