# Setup Workflow (Reusable Pattern)

Use this as a lightweight workflow for launching and evolving a structured project.

## 1) Define Direction
- Set a clear project goal and MVP scope.
- State what is in scope now vs deferred.
- Identify success criteria and risks early.

## 2) Establish Repo Foundations
- Create baseline docs: `README`, decision log, contribution guide.
- Set branch/PR conventions and naming strategy.
- Add labels, issue templates, and (optionally) a project board.

## 3) Start a Living Task System
- Keep one working todo document as the current execution baseline.
- Convert actionable items into issues.
- Group issues by milestone/phase.

## 4) Run a Research → Reflection Loop
- Review relevant prior work and constraints.
- Capture findings in short reflection notes.
- Record open questions and next actions in each note.

## 5) Converge on Decisions
- Consolidate reflections into explicit architectural/design decisions.
- Document rationale and trade-offs.
- Mark superseded ideas and keep one current source of truth.

## 6) Plan Phased Execution
- Split work into phases (setup, MVP, iteration).
- Define exit criteria per phase.
- Keep implementation tasks separate from exploration notes.

## 7) Execute with Review Gates
- Work in short-lived branches.
- Use PRs to review scope, links, and decision impact.
- Keep canonical/main branch protected from unreviewed changes.

## 8) Keep Documentation in Sync
- Update README/todo/decisions as project truth changes.
- Fix broken links and stale references quickly.
- Ensure docs reflect current workflow, not planned ideals.

## 9) Maintain a Devlog Cadence
- Publish concise milestone records (what changed, why, what’s next).
- Capture both outcomes and workflow lessons.
- Use devlogs to make future project replication faster.

## 10) Repeat as an Iterative Cycle
- Reassess priorities at each milestone.
- Promote validated patterns to standards/templates.
- Continue: explore → decide → implement → review → document.
